# Этап 1: Проектирование требований

**Недели:** 3–4 | **Вес:** 10%

## Use Case — Диаграмма вариантов использования

Система включает 11 прецедентов для 4 акторов: Читатель, Библиотекарь, Администратор, Платёжная система. Ключевые отношения: BorrowBook →extend→ PayFine (нельзя взять книгу при штрафах), ReturnBook →include→ PayFine (штраф при просрочке), BorrowBook →include→ ReserveBook (бронь снимается при выдаче).

![Use Case диаграмма](../images/use_case_diagram.png)

## Domain Model — Диаграмма классов предметной области

Модель содержит 6 сущностей (User, Book, BookInstance, Loan, Reservation, Fine), 1 объект-значение (Author) и 4 перечисления статусов. Книга является корнем агрегата для экземпляров. Выдача порождает штраф при просрочке.

![Domain Model](../images/domain_model.png)
