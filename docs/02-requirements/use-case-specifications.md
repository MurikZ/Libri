# Спецификации прецедентов

## UC-004: Взять книгу

| Поле | Значение |
|---|---|
| **ID** | UC-004 |
| **Акторы** | Читатель (инициатор), Библиотекарь (исполнитель), Система |
| **Предусловия** | Читатель аутентифицирован; книга AVAILABLE; штрафов нет; активных выдач < 5 |
| **Постусловия** | Создан Loan(ACTIVE); BookInstance.status = ON_LOAN; активная бронь снята |

**Основной поток:**
1. Библиотекарь находит книгу по названию, ISBN или инвентарному номеру
2. Система отображает информацию о книге и доступных экземплярах
3. Библиотекарь вводит идентификатор читателя
4. Система проверяет: наличие штрафов, лимит < 5, статус книги AVAILABLE
5. Система создаёт Loan(ACTIVE), dueDate = today + 30 дней
6. Система обновляет BookInstance.status → ON_LOAN
7. Система уведомляет читателя

**Исключения:**
- E1: лимит 5 книг достигнут — отказ с сообщением об ошибке
- E2: неоплаченный штраф — переход к UC-006 (Оплатить штраф)
- E3: книга недоступна — предложение UC-011 (Забронировать)

## UC-005: Вернуть книгу

| Поле | Значение |
|---|---|
| **ID** | UC-005 |
| **Акторы** | Библиотекарь (исполнитель), Система |
| **Предусловия** | Существует активная запись Loan для данного экземпляра |
| **Постусловия** | Loan.status = RETURNED; BookInstance.status = AVAILABLE; Fine создан при просрочке |
| **Формула штрафа** | amount = дней_просрочки × 5 руб. (не более 100% стоимости книги) |

**Основной поток:**
1. Библиотекарь вводит инвентарный номер экземпляра
2. Система находит активную запись Loan
3. Система сравнивает returnDate с dueDate
4. Если просрочка — автоматически создаётся Fine
5. Система обновляет Loan.status → RETURNED
6. Система обновляет BookInstance.status → AVAILABLE

**Исключения:**
- E1: книга повреждена — статус DAMAGED, штраф на ремонт
- E2: книга числилась утерянной — уведомление администратора
