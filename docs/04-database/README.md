# Этап 3: Проектирование базы данных

**Недели:** 7–8 | **Вес:** 10%

## ER-диаграмма (логическая модель)

База данных содержит 8 таблиц: users, books, authors, book_authors (M:N), book_instances, loans, reservations, fines. Нормализована до 3НФ. Ключевые связи: один пользователь — много выдач; одна книга — много экземпляров; одна выдача — один штраф (1:1).
