# Бизнес-модель

## Контекст

**Libri** — мобильная информационная система для автоматизации работы публичной или учебной библиотеки.

## Стейкхолдеры

| Стейкхолдер | Роль | Ключевые интересы |
|---|---|---|
| Читатель | Конечный пользователь | Быстрый поиск, бронирование, просмотр истории |
| Библиотекарь | Оператор | Учёт выдач, управление каталогом |
| Администратор | Администратор | Управление пользователями, отчётность |

## Глоссарий

| Термин | Определение |
|---|---|
| Книга (Book) | Библиографическая единица с ISBN, автором, годом издания |
| Экземпляр (BookInstance) | Физический экземпляр книги с инвентарным номером |
| Выдача (Loan) | Факт выдачи экземпляра читателю на срок 14 дней |
| Бронь (Reservation) | Резервирование книги для читателя на 3 дня |
| Штраф (Fine) | Денежное взыскание за просрочку (5 руб./день) |
| Просрочка | Состояние выдачи, когда текущая дата > dueDate |

## Бизнес-процессы

1. **Поиск книги** — читатель ищет по названию, фильтрует по статусу
2. **Бронирование** — читатель бронирует доступную книгу на 3 дня
3. **Выдача** — библиотекарь оформляет выдачу конкретного экземпляра
4. **Возврат** — библиотекарь принимает книгу, при просрочке создаётся штраф
5. **Оплата штрафа** — читатель оплачивает штраф через приложение
